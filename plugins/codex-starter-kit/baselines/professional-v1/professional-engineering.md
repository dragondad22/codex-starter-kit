# Professional engineering baseline v1

**Identity:** `baseline:professional-engineering:v1`

**Version:** `1.0.0`

**Source authority:** `DEC-0017`

This is the offline plugin projection of the approved universal professional-engineering
baseline. DEC-0017 in the source repository remains authoritative. This projection guides
the create review; it is not a signed policy pack, verified control result, legal or
regulatory opinion, or permission to weaken project/workspace authority.

## Universal outcomes

Every project is expected to preserve, in proportion to actual risk:

- explicit scope, ownership, decisions, assumptions, and limitations;
- reviewable changes connected to executable work;
- deterministic tests and validation at the highest stable interface seam;
- truthful failure, coverage, not-applicable, not-configured, needs-review, and accepted-
  exception states;
- least authority, secret protection, dependency/provenance awareness, and no silent data
  transmission or tool activation;
- native Linux, macOS, and Windows behavior for universal product capabilities;
- distinct human-owned records, generated projections, and machine state;
- recovery, reconciliation, evidence, documentation, and change records proportionate to
  the work; and
- accessible, audience-appropriate communication without turning ceremony into assurance.

## Variable engagement

Project profile and human preference may change prompts, explanation depth, batching,
review ceremony, and evidence presentation. They do not remove the universal outcomes,
convert missing evidence to pass, invent approval, or broaden handling/tool authority.

## Seed create limitations

The current create-v1 engine seeds a managed repository contract, approved brief, confirmed
owner persona, stable routes, ownership/provenance, a `not_configured` policy lock, and an
initial conformance projection that reports no verified controls. The special-data
declaration is not persisted by create-v1. Secret scanning, signed policy distribution,
verified packaged-engine provenance, and aggregate conformance remain unavailable or
explicit non-pass until their governing work supplies evidence.

## Review rule

The create workflow must display these limitations with the immutable plan. Applying the
plan establishes only the engine's returned repository lifecycle result. It does not prove
that every baseline outcome or control passes.
