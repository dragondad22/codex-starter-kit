# Codex Starter Kit — Documentation

## Current authority

1. `discovery/CODEX_STARTER_KIT_REVIEW.md` — approved D1–D12 decisions and critique
   history. It governs if a derived document conflicts.
2. `product/PRD.md` — synthesized product requirements and success measures.
3. `product/PERSONAS.md` — authoritative human audiences, motivations, perspectives,
   constraints, authority, and communication needs.
4. `architecture/ARCHITECTURE.md` — system modules, interfaces, adapters, contracts,
   security, transactions, and versioning.
5. `architecture/LIFECYCLES.md` — repository, issue, control, release, upgrade, and risk
   transitions.
6. `architecture/POLICY_PACKS.md` — initial policy ownership and regulatory-pack method.
7. `roadmap/IMPLEMENTATION_ROADMAP.md` — implementation sequence until GitHub becomes the
   live roadmap authority.

## Document lifecycle

These build documents are drafts derived from approved discovery. Before implementation,
the repository-governance milestone will promote durable architectural choices into
stable decision records, configure GitHub Issues/Projects, and replace hand-maintained
roadmap status with the live Horizon view.
