# Security Policy

## Supported versions

The project is not yet production-ready and has no supported release. Security reports
about repository automation, design flaws, or future implementation are still welcome.

## Reporting a vulnerability

Do not open a public issue containing vulnerability details, credentials, exploit code,
private data, or sensitive evidence.

Use GitHub's private vulnerability reporting for this repository when available. If that
surface is unavailable, contact the repository owner through the private contact method
listed on the GitHub profile and include only enough initial detail to establish a secure
follow-up channel.

Include affected revision/version, impact, reproduction conditions, and any known
mitigations. Never include live credentials; revoke them first when possible.

## Disclosure

The maintainer will acknowledge the report, establish severity and scope, coordinate a
fix and disclosure plan, and credit reporters who want attribution. Timelines depend on
impact and whether upstream tools or policy packs are involved.
